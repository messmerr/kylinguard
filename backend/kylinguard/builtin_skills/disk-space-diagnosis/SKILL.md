# 目标

定位空间压力来自哪个分区、目录和文件，给出可复核的容量证据。诊断默认只读，不能把“需要释放空间”自动解释成删除授权。

# 工作流

1. 先调用 `sysinfo.disk_usage` 确认各挂载点使用率与可用空间。
2. 对异常挂载点调用 `disk.disk_hotspots`，从较浅深度开始逐级缩小范围，不要无目的地扫描整机。
3. 对已确认的热点目录调用 `disk.large_files`，列出候选文件；说明哪些只是占用较大，哪些可能安全清理，不要仅凭文件大小断言可以删除。
4. 只有在管理员明确要求清理、目标是普通文件且已展示证据时，才可以提出 `disk.clean_file`。实际删除仍必须经过 KylinGuard 的风险门控与人工确认。
5. 如果空间正常但用户报告卡顿，可使用 `disk.io_stats` 区分容量问题和 I/O 压力。

# 输出要求

报告异常挂载点、主要热点、候选大文件和建议释放量。没有获得删除授权时，只给建议，不执行任何修改。
