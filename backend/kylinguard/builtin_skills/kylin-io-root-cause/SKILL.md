# 适用范围

用于“磁盘很慢”“写入卡顿”“I/O 等待高”“设备繁忙”“是不是磁盘满了”等问题。容量与性能必须分别验证：磁盘接近满可能影响性能，但空间充足不能排除 I/O 异常，I/O 繁忙也不能证明容量不足。

# 强制原则

1. 先判断问题是容量、I/O 性能还是两者都有，再选择后续工具。不得把 `df` 使用率当成设备利用率，也不得把 `%util` 当成分区容量。
2. 优先使用 `iostat` 的有时间窗口采样。降级到 `/proc/diskstats` 时，只能报告累计计数和设备是否出现，不能编造 await、吞吐率、IOPS 或 `%util`。
3. 单次采样描述的是该采样窗口。没有重复采样、历史基线或明确负载背景时，不得把瞬时峰值写成持续故障。
4. 分区、LVM/DM、软件 RAID、物理盘和挂载点不是同一层。必须先从 lsblk 拓扑确认映射，不能用挂载点名称猜物理设备。
5. 诊断默认只读。不得执行卸载、格式化、fsck 修复、RAID 变更、调度器修改、缓存清理或删除文件。
6. 阈值只用于排查优先级，不作为跨设备的绝对故障标准。延迟是否异常需要结合设备类型、业务 SLO 和历史基线。

# 工作流

## 1. 确认平台与范围

调用 `kylin.system_identity` 记录系统和架构。询问或提取：

- 受影响挂载点、应用或设备；
- 读慢、写慢还是双向；
- 问题开始时间和是否持续；
- 是否存在批处理、备份、日志突增等负载变化。

只有用户或现有证据给出具体设备时才向 `kylin.io_diagnosis` 传 device；否则先采集所有块设备，不猜 `/dev/sda`。

## 2. 分离容量问题

调用 `sysinfo.disk_usage`，记录相关挂载点的总量、已用、可用和使用率：

- `>=95%`：严重容量压力，优先处理但仍不能替代 I/O 诊断；
- `85%–94%`：高容量压力；
- `<85%`：通常不足以单独解释“磁盘满”，继续 I/O 分支。

仅当容量异常且需要定位空间来源时调用 `disk.disk_hotspots`。从相关挂载点、`depth=1` 开始，必要时逐层收敛；不要无目的扫描整个 `/`，也不要把大目录直接写成可删除目录。

如果容量正常，明确写“当前没有容量压力证据”，不要为了调用工具继续热点扫描。

## 3. 建立设备拓扑与 I/O 证据

调用 `kylin.io_diagnosis`，先阅读：

- `scope`：全设备还是具体设备；
- `topology`：分区、LVM/DM、RAID、物理盘、文件系统和挂载点；
- `io_source`：`iostat` 还是 `proc_diskstats`；
- `io_evidence` 和 `limitations`；
- 每项 evidence 是否成功、是否截断；
- 是否只探测到麒麟原生工具而未实际调用。

把挂载点沿拓扑映射到实际承载设备。映射不清时停止设备级归因，标为“拓扑未确认”。

## 4. 解释 iostat 采样

仅在 `io_source=iostat` 且表头/设备行完整时解释：

- `r/s`、`w/s`：采样窗口 IOPS；
- `rkB/s`、`wkB/s` 或等价列：吞吐；
- `await` 及读写 await：平均请求等待时间；
- `aqu-sz`：平均队列深度；
- `%util`：设备在窗口内的忙碌比例。

结论规则：

- 高 await + 队列增长：支持“请求排队/设备响应慢”，但仍需结合负载；
- 高 `%util` + 高吞吐且延迟符合基线：可能是繁忙但健康，不能直接写故障；
- 低 `%util` 不一定排除网络盘、RAID、DM 或上层锁竞争；
- 某一设备异常必须引用该设备行，不能把 avg-cpu 或其他盘的数据套过来；
- 输出缺列、被截断或设备未出现时，不做指标推断。

## 5. 解释降级证据

当 `io_source=proc_diskstats`：

- 可以确认设备存在以及取得了内核累计计数；
- 可以比较同一设备两个明确时间点的计数差，但本轮只有一个点时不能计算速率；
- 不得写具体 IOPS、MB/s、await、队列时间或利用率；
- 输出为空时说明设备名可能不匹配或权限/平台证据不足。

## 6. 根因分级

分别报告：

- **容量根因**：相关挂载点高使用率且热点证据能够解释；
- **设备 I/O 根因**：设备映射明确，采样指标与问题窗口一致；
- **高可能上层原因**：设备指标正常但应用仍慢，可能是文件锁、数据库内部等待、网络存储或应用队列，必须标为未验证；
- **历史/瞬时异常**：单次窗口看到峰值但没有持续性证据；
- **证据不足**：只取得累计计数、拓扑缺失或输出截断。

# 停止条件

- 设备或挂载点无法建立映射：停止物理设备归因；
- 仅有 `/proc/diskstats` 单点：停止速率、延迟和利用率结论；
- 容量正常且没有 I/O 指标异常：停止“磁盘故障”断言，建议转向应用/文件系统层；
- 用户只要求诊断：不清理文件、不卸载、不修改存储配置；
- 需要破坏性检测或离线 fsck：停止并说明需要独立维护窗口和授权。

# 输出格式

1. **结论摘要**：容量、I/O、两者都有、未发现证据或证据不足；
2. **容量证据**：相关挂载点和使用率；
3. **设备拓扑**：挂载点到块设备的映射；
4. **采样窗口与指标**：注明 iostat 或累计计数降级；
5. **根因分级**：已确认、高可能、排除和未确认；
6. **证据限制**：单次采样、缺列、截断、权限或工具降级；
7. **下一步验证**：复测时间窗、业务关联或独立变更建议。
